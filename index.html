關於爬蟲

網際網路搜尋引擎網站或是提供資訊檢索服務的公司，系統背後通常會設計一個網路漫遊器，自動到各大網站上擷取網頁資訊，抓取大量的網頁後，在這些網站上建立索引後提供搜尋服務，像是Google, Yahoo等。

模擬人類瀏覽網頁行為，週期性持續性自動化拜訪網站，從各網站擷取部分或全部網頁內容的程式稱之為網路爬蟲（Web Crawler）或網路蜘蛛（Web Spider）程式。

網頁選擇介面

  1. 點擊Chrome中 圖示，然後選「新增任務」，出現以下設定畫面：
  2. 先進到欲監控的頁面http://www.mobile01.com/topiclist.php?f=638 打開編輯畫面，「任務名稱」可以隨使用者喜好設定，如：Mobile 01。監控起始頁面可貼上現在的網址http://www.mobile01.com/topiclist.php?f=638，這將成為擷取網頁的起始點。可以按填入標題和填入網址自動填入任務名稱和網址。
  3. 爬行網頁和目標網頁需要填入符合Python語法的正規表示式，有關於正規表示式的說明，會另提供解說資料，不在本文件討論範圍之內。其作用只要是限制爬行的範圍，目前的設定可確保不會爬到別的網站或是別的討論主題。 爬行網頁請填入：^http://www\.mobile01\.com/topiclist\.php\?f=638 目標網頁請填入：^http://www\.mobile01\.com/topicdetail\.php\?f=638
  4. 一旦目標網頁填寫完畢後，使用者會發現下面多出一行紅色的警告標語(此網頁非目標頁面！)。 這是因為並非真的要擷取此看板網頁的內容，要擷取的是應該是討論串裡面的內容。使用者可以隨意進入看板內任一討論串：http://www.mobile01.com/topicdetail.php?f=638&t=3922961&last=50328768 此時警告標語已經消失，若沒有，需檢查是否目標網頁設定有問題。接下來，請按新增區塊，來明確指定要擷取的網頁區塊。

主控台介面

  1. 修改系統使用者： 使用者點選畫面上方「Select user to change」頁籤，出現使用者名稱及基本資料，可點擊名稱進行修改。
  2. 設定系統使用者狀態群組： 使用者進入編輯頁面後，可在「區域A」勾選欲設定之使用者狀態，在「區域B」設定使用者所屬群組。

  • Client: Start Bootstrap
  • Date: April 2014
  • Service: Web Development

區塊選擇介面

  1. 接下來，請按新增區塊，來明確指定要擷取的網頁區塊。
  2. 選定某區塊後，按左鍵確定，下方會出現確定或取消的按鈕。並且所有同樣路徑的區塊會全部被標記為紅色的背景。 確認選取範圍後，請按確定，否則請按取消。
  3. 按下確定後，再回到編輯介面，使用者會看到新的區塊已經新增上去，使用者可以進一步編輯區塊名稱，方便日後整理或分析資料。使用者也可以選擇擷取本文(預設值)、超連結、檔案資源或是HTML原始碼。
  4. 最後，使用者可以繼續新增區塊，或是按儲存完成編輯。

  • Client: Start Bootstrap
  • Date: April 2014
  • Service: Web Development

使用流程

  1. 設定爬蟲任務。.
  2. 爬蟲任務清單與狀態：提供爬蟲的狀態包括：活躍/啟動、 停止作業、數據量異常、 "No Data" 錯誤信息、提供email alert，具備警示功能，如目標網站無法取得資料將發出通 知。
  3. 爬蟲時序(Timer)管理：設定爬蟲數據採集時段ex：每x 小時啟動。
  4. 爬蟲Latency(延遲控制)：預防對資料頻道過分數據採集，設定爬蟲"Crawler Request"的時間區間（秒）。
  5. VPN服務設定：啟動VPN設定頁面以方便設定 Proxy List。
  6. 爬蟲任務保存設定：保存爬蟲設定功能，符合權限的使用者可參考和輕易修改更新現有爬蟲設定。
  7. 爬蟲任務啟動和停止：使用者可隨時啟動和停止指定爬蟲任務。

  • Client: Start Bootstrap
  • Date: April 2014
  • Service: Web Development

爬蟲流量控制

  1. 區域A：爬蟲過濾器 使用者可以選擇要爬蟲軟體監控網站哪些活動，並設定過濾器名稱： ‧ 選擇「全部(All)」，代表監控該網站的所有活動。 ‧ 選擇「活躍(Active)」，代表只監控網站最活躍的活動。 ‧ 選擇「不活躍(Inactive)」，代表只監控網站一般性活動。 ‧ 選擇「有問題(Problem)」，代表監控網站異常的活動。
  2. 區域B：爬蟲狀態趨勢分析 分析爬蟲在不同時間內各網站抓取狀態，看出在特定時間內監控網站是否有異常，使用者可選擇所欲分析時間，點選下拉框會出現「過去24小時 內」、「30天內」、「一年內」三種時間選項。
  3. 區域C：監控任務面板 使用者設定過濾器名稱後，監控任務面板下方會顯示該名稱，使用者 可點擊「開始」按鈕(綠色三角形圖示)，會出現監控起始時間、該網站抓爬資料量、前一次更新時間、以及過去24小時抓爬的資料總量。
  • Client: Start Bootstrap
  • Date: April 2014
  • Service: Web Development

爬蟲參數設定

使用者可設定要抓爬的主要網頁、種子網頁、連結頁面、目標頁面，設定要抓爬的網頁內容條件，並設定當爬蟲抓爬數量低於設定值時，由系統發出異常警示e-mail。
